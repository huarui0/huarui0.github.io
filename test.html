          1. Change location to the top-level directory of your MySQL installation, which is typically /usr/local/mysql (adjust the path name for your system as necessary): 
            cd /usr/local/mysql
          2. Create a directory whose location can be specified as the value of that variable: - The secure_file_priv system variable limits import and export operations to a specific directory. 
            mkdir mysql-files
            Grant directory user and group ownership to the mysql user and mysql group, and set the directory permissions appropriately: 
            chown mysql:mysql mysql-files
chmod 750 mysql-files
          3. Use the server to initialize the data directory, including the mysql schema containing the initial MySQL grant tables that determine how users are permitted to connect to the server. For example: 
            bin/mysqld --initialize --user=mysql
          4. If you want to deploy the server with automatic support for secure connections, use the mysql_ssl_rsa_setup utility to create default SSL and RSA files: 
            bin/mysql_ssl_rsa_setup
            Note: The mysql_ssl_rsa_setup utility is deprecated.
          5. In the absence of any option files, the server starts with its default settings.
            To explicitly specify options that the MySQL server should use at startup, put them in an option file such as /etc/my.cnf or /etc/mysql/my.cnf.

    • ✨ 说明 ✨
      • 针对 Binary or Package-based 安装 的 MySQL，有 三种 的 升级 方式：
        • In-place upgrade 
          最简单 的 升级 方式。
如果是复杂的系统，如 Cluster 级别的，需要达到 专业 水平，就要掌握 更全面、复杂 的 升级方法，
否则，只要 先掌握 In-place upgrade 就可以。
        • Logical Upgrade
        • MySQL Cluster Upgrade
      • In-place upgrade 的 说明
        • In-place upgrade，实质上 就是 用 新 安装包 替换 旧 安装包，然后 再 将 新系统 的 数据目录，指向 旧系统 在用的 数据目录，然后 再更新 需要更新 的 部分。
          最直接的办法就是：将 旧系统 的 配置文件（my.cnf） 直接 拷贝 到 新系统 的 $MYSQL_HOME/ 目录下（如果 有 设置 $MYSQL_HOME，则 需要 将 路径 更新 到 新系统的 路径 上。
        • 将 新系统 的 数据目录，指向 旧系统 在用的 数据目录，有两种方法：
          • 方法 2: 将 旧系统 的 配置文件（my.cnf） 直接 拷贝 到 新系统 的 $MYSQL_HOME/ 目录下 -- 推荐 的 方法 ❕
          • 方法 2: 直接 在 启动 命令中，指定 旧系统 的 数据 目录。
            mysqld_safe --user=mysql --datadir=/path/to/existing-datadir &
      • Logical Upgrade 的 说明
        • Logical Upgrade，实质上 就是 安装 一套 新系统，然后 初始化，再 将 旧系统 的 数据 导出，然后 用导入 的 方式，将 数据 导入 新系统，如果 数据量 比较大，是 一个挑战。。。
